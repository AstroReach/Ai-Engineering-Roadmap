# Solo Hackathon Roadmap

**Goal:** One person, AI-assisted workflow, ship fast and win hackathons.


## Core Mental Model

| Role            | Responsibility                      |
| --------------- | ----------------------------------- |
| **You**         | Product Manager + Director          |
| **Claude Code** | Full-Stack Engineer + Debugger      |
| **Tools**       | Handle infrastructure automatically |

### Workflow Breakdown

* You describe what to build
* Claude writes and debugs code
* Tools handle deployment and scaling
* You read errors and guide direction

---

# Phase 1 — Foundations (Week 1–2)

**Objective:** Learn core fundamentals that apply everywhere

| Topic               | Why It Matters                              |
| ------------------- | ------------------------------------------- |
| Git + GitHub        | Version control, collaboration, submissions |
| JavaScript (Basics) | Understand frontend code                    |
| Python (Basics)     | Understand AI/ML code                       |
| APIs (REST + JSON)  | Communication between frontend & backend    |
| Command Line        | Run scripts, navigate systems               |

---

# Phase 2 — Web App Layer (Week 3–4)

**Objective:** Build and ship functional apps quickly

| Layer           | Technology             | Depth Required           |
| --------------- | ---------------------- | ------------------------ |
| Frontend        | Next.js + Tailwind CSS | Basics                   |
| UI Components   | Shadcn/UI              | Usage only               |
| Backend         | Next.js API Routes     | Basics                   |
| Database + Auth | Supabase               | Basics                   |
| Deployment      | Vercel                 | Minimal (push to deploy) |

---

# Phase 3 — AI Layer (Week 5–6)

**Objective:** Add intelligence (your competitive advantage)

| Area             | Technology              | Depth Required       |
| ---------------- | ----------------------- | -------------------- |
| LLM Integration  | Claude API / OpenAI SDK | Strong understanding |
| AI Orchestration | LangChain               | Basics               |
| Vector Search    | Supabase pgvector       | Basics               |
| RAG Systems      | LlamaIndex              | Basics               |
| Backend (AI)     | FastAPI (Python)        | Basics               |

---

# Phase 4 — AI / ML Layer (Week 7–8)

**Objective:** Stand out with advanced capabilities

| Area              | Technology     | Depth Required |
| ----------------- | -------------- | -------------- |
| Data Handling     | Pandas + NumPy | Basics         |
| Pretrained Models | Hugging Face   | Usage          |
| Image AI          | Replicate API  | Usage          |
| Speech-to-Text    | Whisper API    | Usage          |
| ML Fundamentals   | Scikit-learn   | Concepts       |

# Priority Learning Order

Follow strictly — don’t jump ahead.

1. Git + GitHub
2. JavaScript basics
3. APIs (REST + JSON)
4. Next.js + Tailwind
5. Supabase
6. Claude API (deep focus)
7. Python basics
8. FastAPI
9. LangChain / LlamaIndex
10. Hugging Face

---

# 8-Week Learning Timeline

| Week | Focus                              | Goal               |
| ---- | ---------------------------------- | ------------------ |
| 1–2  | Git + JS + Python                  | Strong foundation  |
| 3–4  | Next.js + Tailwind + Supabase      | Build basic app    |
| 5–6  | Claude API + LangChain + FastAPI   | Add AI features    |
| 7–8  | Hugging Face + Replicate + Whisper | Add advanced AI    |
| 9+   | Build projects + join hackathons   | Ship real products |

# 24-Hour Hackathon Workflow

| Time       | Action                                    |
| ---------- | ----------------------------------------- |
| Hour 1     | Pick idea + scaffold app + setup Supabase |
| Hour 2     | Generate UI + push to GitHub              |
| Hour 3–5   | Build core feature                        |
| Hour 6–8   | Add AI integration                        |
| Hour 9–10  | Deploy (Vercel)                           |
| Hour 11–24 | Polish, debug, prepare demo               |

---

# Key Insight

Focus on **concepts, not syntax**.

* Understand *what* needs to be built
* Let AI tools handle *how*
* Your value = direction, decisions, and product thinking
